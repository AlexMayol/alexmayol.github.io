Let my first tell you my story with Tailwind:

The first time I heard about it and visited the site, I definitely didn't like the way it works, so I decided to write my own CSS framework to remedy all the problems I found on my daily layout work.

I then found myself doing the exact same thing Tailwind does, so I decided to give it a try and I absolutely fell in love with it. I know this is the general reaction of people, as you often tweet, but did it change the way I build websites...

Ever since then, I've built ALL of my work proyects with it, no exception. Here's the list in case you are interested:

  • Transferencia24, an ecommerce where people can buy our service to interact with the Spanish National Department of Traffic and easy do various administrative managements. My biggest proyect so far. I'm really proud of it.
  • Worldcars, a car dealership website where we list all the stock so people can appoint a visit and try them. I first used my CSS framework I mentioned before, but once I figured the correct PostCSS config for my workflow, I rewrote the whole site with Tailwind.
  • Movehability, online personal trainer website. I had all the needed configs from the other projects so it was Tailwind from the very beginning.

Because the frontend ecosystem has been evolving rapidly in the last years, I'm really enjoying working on sites using flexbox or Grid to build the layout, and writting Vue, which I find is the best fit for me. I'm really excited about the increasingly global coverage of these new features and how it will impact bundle sizes and performance overall. Performance monitoring and optimization in the frontend is my favourite part, and I graduated with a thesis on web accessibility and testing tools, so I've always kept in mind the foundation for a great website is a good markup.

I'm currently working as a full-stack developer (VueJS, PHP) in Spain, where I live, but ever since I started learning web dev (during college), I've loved frontend and it has been my main focus.
